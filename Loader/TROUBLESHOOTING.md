# Устранение проблем с путями

## Проблема: "Папки не найдены"

### Причина
Приложение не может найти папки `DB/ServerBooks/Books/` и `DB/ServerBooks/Images/`.

### Решение

1. **Запустите приложение** и посмотрите логи при старте
2. Проверьте, что в логах отображаются пути:
   - `BaseDir: ...`
   - `BooksPath: ...`
   - `ImagesPath: ...`

3. **Если пути неверные**, переместите Loader.exe в нужное место:
   - Рекомендуется запускать из корня проекта `LIB/`

4. **Альтернативное решение**: Отредактируйте `MainWindow.xaml.cs` и замените:
   ```csharp
   _booksFolderPath = System.IO.Path.Combine(projectRoot, "DB", "ServerBooks", "Books");
   ```
   на:
   ```csharp
   _booksFolderPath = @"C:\Users\User\Documents\GitHub\LIB\DB\ServerBooks\Books";
   ```
   (укажите ваш реальный путь)

## Структура путей

Приложение ищет:
```
LIB/
├── DB/
│   └── ServerBooks/
│       ├── Books/      ← файлы книг (.fb2)
│       ├── Images/     ← обложки (.jpg)
│       └── Текстовый документ.txt
└── Loader/
    └── bin/Debug/net9.0-windows/
        └── Loader.exe  ← запускается отсюда
```

## Отладка

Чтобы увидеть точные пути, запустите приложение и проверьте логи в окне.
Логи отображаются автоматически при старте приложения.
